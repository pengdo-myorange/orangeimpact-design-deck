# 슬라이드 작성 규칙 — im-not-ai 통합

출처: [epoko77-ai/im-not-ai](https://github.com/epoko77-ai/im-not-ai) (한국어 AI-티 제거 분류 체계 v2.0)

**적용 시점**: 슬라이드 본문·헤드라인·스피커 노트·CTA 카피·이미지 캡션 **초안을 작성하는 순간부터** 모든 한국어 텍스트에 적용. 사후 윤문하지 않는다.

---

## 4대 절대 원칙

1. **의미 불변**: 사실·주장·숫자·이름·직접 인용은 100% 보존한다.
2. **근거 기반**: 감지된 패턴에 한해 수정한다.
3. **장르 보존**: 칼럼을 에세이로 바꾸지 않는다.
4. **과도한 인간화 금지**: 같은 의미를 다른 결로 쓸 뿐, 임의의 정서·수사를 덧붙이지 않는다.

---

## 카테고리별 작성 규칙

### A — 번역투 (S1 / 항상 회피)
- ✗ "~를 통해", "~에 대해", "~에 있어서"
- ✗ 이중 피동 ("~되어지다", "~가 ~로 보여진다")
- ✗ "그/그녀" 3인칭 대명사 남용 (한국어는 주어 생략이 자연스럽다)
- ✗ 좌분지 관계절 과다 ("내가 어제 만난 그 사람이 말한 이야기에 따르면…")
- ✓ "~로", "~을(를)", 능동형, 주어 생략

### B — 영문 병기 절제
- ✗ 모든 전문 용어마다 괄호로 영문 병기 — 첫 등장에만, 필요할 때만
- ✓ 한국어로 충분히 통하는 용어는 영문 생략

### C — 구조 패턴 (S1)
- ✗ 기계적 "첫째 / 둘째 / 셋째" 나열
- ✗ 모든 항목 앞에 콜론 헤더 (":" 으로 끝나는 라벨)
- ✗ 슬라이드마다 같은 불릿 개수·동일한 들여쓰기
- ✓ 항목 수와 구조를 내용에 맞게 변주

### D — AI 특유 문구 (S1 / 절대 금지)
- ✗ "결론적으로", "시사하는 바가 크다", "주목할 만하다"
- ✗ "본 발표에서는…", "본 자료는…"
- ✗ 과장 어휘 ("혁신적인", "획기적인", "전례 없는")
- ✓ 구체적 사실로 대체

### E — 리듬 (S2)
- ✗ 모든 문장 길이가 비슷함 (분산 < 5단어)
- ✗ 동일 종결어미 연속 ("…입니다. …입니다. …입니다.")
- ✓ 짧은 문장과 긴 문장 교차, 종결 변주

### F — 과한 수식 (S2)
- ✗ "-적/-성/-화/-tion/-ment" 남용
- ✗ 형용사 연쇄 ("매우 중요하고 핵심적이며 본질적인…")
- ✓ 한 단어로 충분한 곳에 한 단어만

### G — 과한 헤지 (S2)
- ✗ 한 구절에 헤지 두 개 이상 ("아마 어쩌면 ~할 수도 있을 것으로 보입니다")
- ✓ 단정할 수 있으면 단정, 불확실할 때만 한 번 헤지

### H — 접속사 남발 (S2)
- ✗ 문장 첫머리 "또한", "따라서", "즉", "나아가" 반복
- ✓ 문장 흐름으로 연결 — 접속사는 진짜 필요할 때만

### I — 형식명사 (S2)
- ✗ "~것이다", "~점이다", "~수 있다", "~바이다", "~할 필요가 있다"
- ✓ 동사·서술어로 직접 표현

### J — 시각 장식 절제 (S2)
- ✗ 한 슬라이드에 **볼드** 다섯 군데 이상
- ✗ 따옴표·줄표·대시로 모든 명사 강조
- ✓ 강조는 한두 곳으로 — 강조가 곧 위계

## 슬라이드 헤드라인 체크리스트

작성 후 자문:
- [ ] 첫 단어가 동사·구체명사인가? (추상명사 시작은 AI-티)
- [ ] 30자 이내인가?
- [ ] D 카테고리 표현이 0건인가?
- [ ] 같은 덱의 다른 헤드라인과 종결 어미가 다른가?

## 본문 체크리스트

- [ ] 한 슬라이드 내 동일 종결어미가 3회 연속되지 않는가?
- [ ] 문장 길이 평균이 비슷하면 일부러 짧은 문장 한두 개 끼웠는가?
- [ ] 영문 병기는 첫 등장에만 있는가?
- [ ] 헤지가 한 문장에 1개 이하인가?

## 빌드 시점 lint (lib/lint.js 자동 검증)

다음 S1 패턴이 발견되면 경고 출력. 강제 수정은 하지 않으나 차기 빌드에서 자동 회피.

- `rule: ai-tell-d` — "결론적으로", "시사하는 바", "본 발표"
- `rule: ai-tell-a-passive` — 이중 피동 ("되어지", "보여지")
- `rule: ai-tell-c-mech` — "첫째", "둘째", "셋째" 동시 등장
- `rule: ai-tell-i-formal` — 형식명사 "것이다/점이다" 3회 이상
